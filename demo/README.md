# Demo Assets

This folder contains demo assets for the GitHub repository.

## Required Files

Please add the following files to this directory:

### 1. Banner Image
- **File**: `banner.png`
- **Recommended size**: 1200x400 pixels
- **Description**: Main banner image for the README
- **Suggestion**: Create a banner showing the extension logo with manga-style artwork

### 2. Screenshots

#### Popup Screenshot
- **File**: `popup-screenshot.png`
- **Recommended size**: 320x500 pixels
- **Description**: Screenshot of the extension popup with controls
- **What to capture**: The popup showing "Ready" status with Start/Pause/Resume/Stop buttons

#### Settings Screenshot
- **File**: `settings-screenshot.png`
- **Recommended size**: 600x800 pixels
- **Description**: Screenshot of the settings page
- **What to capture**: Settings page showing API key field, voice selection, speech rate slider, and capture interval

#### Narrating Screenshot
- **File**: `narrating-screenshot.png`
- **Recommended size**: 1920x1080 pixels
- **Description**: Screenshot showing the extension in action
- **What to capture**: A manga page being displayed with the popup showing "Narrating..." status

### 3. Demo Video

Upload your demo video to GitHub and update the README.md with the correct link:

```markdown
https://github.com/user-attachments/assets/your-video-id-here
```

**Video Guidelines:**
- **Length**: 30-60 seconds
- **Content**: Show the complete flow:
  1. Opening a manga page
  2. Clicking the extension icon
  3. Clicking "Start Narration"
  4. Selecting screen to share
  5. Showing the narration in action with audio
  6. Demonstrating pause/resume controls
- **Format**: MP4, WebM, or MOV
- **Resolution**: 1080p recommended

## How to Add Files

1. Create the screenshots and banner image
2. Save them in this `demo/` folder with the exact filenames listed above
3. Record your demo video
4. Upload the video to your GitHub repository (drag and drop into an issue or PR to get the URL)
5. Update the README.md with the actual video URL

## Tips for Great Screenshots

- Use a clean, uncluttered manga page for demos
- Ensure text is readable
- Show the extension working successfully
- Use consistent styling across all screenshots
- Consider adding annotations or highlights to draw attention to key features
